# Spec: API 调用内容采集与行为分析管道

## Overview

在现有 AI Gateway 的 Rust 代理层增加对请求/响应内容的拦截采集能力，实现短期全文存储 + 长期聚合统计，并通过前端提供行为分析仪表盘。所有采集必须异步、零阻塞、可配置保留期。

## Functional Requirements

### FR-1: 内容采集

- **FR-1.1 请求内容采集：** 拦截 `/v1/chat/completions` 和 `/v1/embeddings` 的请求体，提取 `messages` 数组完整内容
- **FR-1.2 响应内容采集：** 记录流式（SSE）和非流式响应的完整回复内容
- **FR-1.3 文件/附件元数据采集：** 检测请求中的媒体引用（图片、PDF 等），记录文件类型、大小、引用位置
- **FR-1.4 元数据采集：** 记录每次调用的用户 ID、模型名称、Token 用量（输入/输出）、延迟、时间戳、API Key 指纹

### FR-2: 数据存储

- **FR-2.1 全文短期存储：** 原始请求/响应内容存储在 PostgreSQL 的 `call_contents` 表中，保留期可配置（默认 30 天）
- **FR-2.2 自动过期清理：** 后台定时任务扫描过期数据并清理，支持可配置的清理间隔
- **FR-2.3 长期聚合统计：** 超过保留期的原始数据自动聚合到 `daily_usage_stats` 表，按天/用户/模型粒度聚合
- **FR-2.4 全文搜索：** 在保留期内支持对 prompt/response 内容的全文搜索

### FR-3: 分析仪表盘

- **FR-3.1 使用趋势：** 以折线图展示选定时间范围内的日调用量、Token 消耗、费用趋势
- **FR-3.2 用户排行：** Top-N 用户按调用量/费用排序
- **FR-3.3 模型排行：** Top-N 模型按调用量/费用排序
- **FR-3.4 成本归属：** 按用户/模型/部门维度的费用拆分视图
- **FR-3.5 质量指标：** 响应时间分布、错误率、无意义 prompt 检测
- **FR-3.6 CSV 导出：** 所有图表数据支持导出

### FR-4: 安全审计

- **FR-4.1 敏感数据检测：** 对 prompt/response 进行正则匹配，检测疑似泄露的 API Key、密码、身份证号等
- **FR-4.2 异常检测：** 识别高频调用、非工作时间大量调用、单一 IP 异常请求模式
- **FR-4.3 审计告警：** 在仪表盘突出显示异常事件

### FR-5: 配置与管理

- **FR-5.1 保留期配置：** 通过环境变量 `CONTENT_RETENTION_DAYS` 配置全文保留天数
- **FR-5.2 采集开关：** 通过环境变量 `CONTENT_CAPTURE_ENABLED` 全局启用/禁用内容采集
- **FR-5.3 脱敏开关：** 通过环境变量 `CONTENT_MASK_ENABLED` 控制是否对敏感内容脱敏

## Non-functional Requirements

- **NFR-1 零性能影响：** 内容采集必须完全异步，不阻塞代理请求/响应主路径
- **NFR-2 仪表盘加载：** 仪表盘主页在 100K+ 日请求量下加载时间 < 3 秒
- **NFR-3 存储效率：** 全文存储使用压缩格式（JSONB），单条记录不超过原始请求的 120%
- **NFR-4 隐私安全：** 敏感内容在列表视图自动截断，查看全文需二次确认
