# Initial Concept

基于现有的 AI Gateway 项目，扩展功能以收集用户在使用 AI API 时上传的请求内容和返回结果，用于日后的行为分析和使用模式挖掘。

# Product Guide

## 产品定位

AI Gateway — 行为分析扩展模块。在现有企业级 AI API 网关基础上，增加对 API 调用内容（请求和响应）的采集、存储与分析能力，为团队管理者提供安全审计、成本优化、使用趋势和质量评估的一站式数据洞察平台。

## 目标用户

- **团队管理者：** 监控团队 API 使用状况、控制成本、确保合规性

## 核心功能

### 1. 内容采集

- **请求内容采集：** 捕获用户发送给 AI 模型的完整 messages/prompt，包括多轮对话上下文
- **响应内容采集：** 记录 AI 模型的完整回复内容（支持流式和非流式两种模式）
- **文件/附件采集：** 记录上传的图片、PDF 等媒体文件的元数据及内容引用

### 2. 数据存储策略

- **短期全文存储（7-30 天）：** 请求和响应的完整原文保存在高性能存储中，支持实时查询和深度分析
- **长期聚合统计（永久）：** 原始数据过期后自动聚合为统计指标（调用量、Token 用量、响应时间、模型分布等），长期保留供趋势分析

### 3. 分析能力

- **使用趋势：** 按模型、用户、时间维度展示调用量的变化趋势
- **质量评估：** 分析 prompt 与响应的相关性、响应质量评分、错误率监控
- **安全审计：** 检测敏感数据（API Key、密码、个人身份信息）泄露风险，识别异常调用模式（高频查询、非工作时间大量调用等）
- **成本优化：** 按用户/模型/时间维度归因费用，识别浪费调用，推荐更经济的模型替代方案

## 与现有系统集成

- 复用现有的 `usage_logs` 表结构，扩展为包含 content 字段的新架构
- 在 Rust 代理层（`proxy.rs`）增加内容拦截与异步写入管道
- 审计日志（`audit_routes.rs`）与分析数据共享同一数据源
- 管理后台（React 前端）新增"行为分析"仪表盘页面
